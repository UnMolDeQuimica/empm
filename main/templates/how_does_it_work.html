Y esto... ¿Cómo funciona?

Esta molécula... ¿Puede matarme? es una web que utiliza diferentes tecnologías para poder hacer predicciones de las propiedades ADMET (adminitración, distribución, metabolismo, excreción y toxicidad) de una molécula.

Estas predicciones no siempre son precisas y bajo ningún concepto deben usarse como una base de datos real.

A modo de resumen, desde la interfaz web construida con Django podemos introducir un código SMILES que será enviado a RDKit para ser validado y luego a ADMET AI para que haga las predicciones correspondientes. Estas predicciones se almacenan en una base de datos para evitar repetir predicciones anteriores.


ADMET AI

El corazón de Esta molécula... ¿Puede matarme? reside en ADMET AI.

Se trata de un proyecto de código abierto cuya finalidad es ofrecer una herramienta para realizar predicciones de las propiedades ADMET de las moléculas.

ADMET AI es, en esencia, un modelo de Machine Learning (una IA, vamos) entrenada con datos de Therapeutics Data Commons para hacer dichas predicciones.

Se puede utilizar como paquete de python, como hacemos aquí, como web o vía cli.

Para más información, podéis visitar este post y, si sois muy pros, podéis leer este artículo.

Desde aquí, le doy mil gracias al equipo de James Zou por su grandiosa contribución.


RDKit

Uno de los engranajes que hacen funcionar a ADMET-AI es RDKit, software de quimioinformática de código abierto escrito en C++ y con una API para trabajar con Python.

Gracias a RDKit, ADMET-AI puede hacer muchos de sus cálculos.

Por mi parte, uso RDKit para procesar los códigos SMILES de las moléculas, generar las representaciones en 2D de las moléculas y calcular los índices de similaridad, aunque ofrece tropomil funcionalidades más.


SQLite

Hacer una predicción con ADMET-AI tiene un cierto coste computacional. Es verdad que no es un coste extremadamente alto, pero nunca viene mal buscar formas de disminuir el uso de CPU.

Una opción es hacer una predicción y almacenarla en una base de datos. Aquí es donde entra SQLite.

Es un motor de SQL ligero ideal para un proyecto chiquitito como este.


Django

Ahora que tenemos las predicciones y las guardamos en una base de datos, necesitamos una página web con la que interaccionar.

Django nos permite usar el lenguaje de programación Python (el mismo con el que nos comunicamos con ADMET-AI) para crear páginas web como esta y comunicarnos con la base de datos a través de su ORM.

Si bien hay alternativas como FastAPI, FastHTML o Flask, mi experiencia en Django es mucho mayor y hacer cosillas como esta me resulta cómodo y sencillo.


HTMX

Para conseguir hacer uso de todas las funcionalidades de la web sin tener que recargar cada vez que hacemos click en un botón o en un enlace, podemos crear una SPA (Single Page Application) y hacer uso de los mil y un frameworks de JS que existen en la actualidad.

Pero lo siento, yo soy desarrollador backend. Todo lo que sea JavaScript da urticaria.

Así que tiro de HTMX, un maravilloso framework que ofrece una experiencia similar a las de las SPA pero usando hipermedia como base.

Fácil, sencillo y para toda la familia.
